# PHASE 9

## User Model

Create a lightweight user model.

Track only information relevant to collaboration.

Examples:

Current objectives

Active projects

Preferred tools

Working habits

Frequently used providers

Coding stack

Do NOT infer sensitive personal attributes.

Allow users to inspect, edit, and delete stored information.

The user model should remain transparent and under user control.

# ENGINEERING RULES

No orphan entities.

No duplicate nodes.

Every edge must have meaning.

Every node must have provenance.

Keep graph updates incremental.

Never rebuild the entire graph unless explicitly requested.
